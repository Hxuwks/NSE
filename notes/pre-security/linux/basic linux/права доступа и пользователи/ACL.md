Перед тем как разобраться что такое acl надо понять теорию рассширенных прав, и что они из себя представляют.

---
#### Специальные права доступа (Special Permissions)
Стандартные права (rwx) дополняются тремя специальными битами. В числовом виде они записываются как четвертая (первая слева) цифра: например, `4755`.

**SUID (Set User ID) — 4000**
- **Для кого:** Только для исполняемых файлов.
- **Эффект:** Программа выполняется с правами **владельца** файла, а не пользователя, который её запустил.
- **Пример:** Команда `passwd`. Чтобы сменить пароль, нужно записать данные в `/etc/shadow`, куда есть доступ только у root. Благодаря SUID обычный пользователь может это сделать.
- **Символ:** `rws------` (буква **s** на месте **x** владельца).
**SGID (Set Group ID) — 2000**
- **Для файлов:** Программа выполняется с правами группы владельца.
- **Для директорий:** Самый частый сценарий. Все файлы, созданные внутри такой директории, автоматически получают ту же **группу**, что и у самой директории (а не основную группу пользователя).
- **Пример:** Общая папка для отдела разработки.
- **Символ:** `rwxrws---` (буква **s** на месте **x** группы).

**Sticky Bit (Липкий бит) — 1000**
- **Для директорий:** Запрещает удаление или переименование файлов внутри директории всем, кроме **владельца конкретного файла** или root.
- **Пример:** Папка `/tmp`. Все могут туда писать, но никто не может удалить чужой временный файл.
- **Символ:** `rwxrwxrwt` (буква **t** в конце).

Работла с acl происходит с помощью команды `setfacl`, которая имеет следующий синтаксис:
```bash 
setfacl [options] operation entity:entityname:permissions file
```
Также можно менять права через простой chmod, но acl предоавтляет больший функционал и широту для применения.
Рассмотрим опции и операции для `setfacl`, которые можно применять для управления правами:
- options : позволяет пр помощи опций управлять работы команды. Основные опции:
	- `-d` : устанавливает acl по умолчанию
	- `-k` : удаляет список acl по умолчанию
	- `-R` : рекурсивно применяет настройки acl
- operation : позволляет добавлять или удалять настройки ACL. Основные операции:
	- `--set` : используется для установки ACL. Она заменяет любой существующий ACL
	- `-m` : дополняет текущие настройки, без замены текущей
	- `-x` : удаляет текущий ACL
- entity и entity name : тип и имя объекта которые определяют для кого надо установить acl. Пользователь -  `u` и группа - `g`
- permissions : устанавливаемые разрешения
- file : имя файла или каталога, к которому необходимо принять изменения
---
После того, как основные опции изучим примеры:

```bash
setfacl -m g:users:rwx /somedir
setfacl -d -m g:users:rwx,g:developers:rx /somedir
```
Для того чтобы получить права файла или каталога используем команду:
```bash
getfacl file
```
