**Процесс** — это программа в состоянии выполнения. У каждого процесса есть уникальный номер — **PID** (Process ID). 

---

1. Мониторинг и поиск процессов

|Команда|Описание и ключи|Практический пример|
|---|---|---|
|`ps`|Статический снимок процессов.|`ps aux \| grep python` — найти все процессы Python.|
|`top` / `htop`|Интерактивный мониторинг.|Запустите `htop`, нажмите `F6` для сортировки или `F9` для убийства.|
|**`pgrep`**|Поиск PID по имени.|`pgrep -u root sshd` — узнать PID процесса sshd от root.|
|**`pstree`**|Дерево процессов (кто кого запустил).|`pstree -p` — показать дерево процессов с их PID.|
|**`lsof`**|Список открытых процессом файлов/портов.|`lsof -i :80` — кто занял 80-й порт (веб-сервер).|

---

2. Завершение процессов (Сигналы)

Для управления процессами используются сигналы. Основные: **15 (SIGTERM)** — просьба закрыться, **9 (SIGKILL)** — немедленное убийство. 

- **`kill <PID>`**: Завершить процесс по ID.    
```bash
    kill 1234          # Мягкое завершение (SIGTERM)
    sudo kill -9 1234  # Принудительное завершение (SIGKILL)
```
    
- **`killall <name>`**: Убить все процессы с одним именем.
```bash
    killall -9 firefox  # Закрыть все окна браузера разом
```

- **`pkill <pattern>`**: Убить по части имени или пользователю.
```bash
    pkill -u alice      # Завершить все процессы пользователя alice
```

3. Приоритеты (Nice и Renice)

Приоритет (Nice) меняется от **-20** (высший) до **19** (низший). По умолчанию — 0. 

- **`nice`**: Запуск с заданным приоритетом.
```bash
    nice -n 10 backup.sh  # Запустить тяжелый бэкап с низким приоритетом
```

- **`renice`**: Изменение приоритета на лету.
```bash
    sudo renice -n -5 -p 1234  # Повысить приоритет важному процессу
```

4. Управление заданиями (Job Control) и фоновый режим

Если вы запустили команду в терминале, её можно отправить в "фон", чтобы терминал освободился. 

1. `Ctrl + Z` — приостановить текущий процесс.
2. **`jobs`** — посмотреть список приостановленных задач.
3. **`bg %1`** — возобновить задачу №1 в фоновом режиме.
4. **`fg %1`** — вернуть задачу №1 на передний план (в фокус).
5. `kill %!` - убить задачу №1
6. **`nohup`**: Запуск процесса, который не "умрет" при закрытии терминала.

```bash
    nohup ./long_script.sh &  # Запуск в фоне с защитой от SIGHUP
```

---

5. Продвинутая диагностика

- **`strace`**: Отслеживание системных вызовов процесса. Помогает понять, почему программа "зависла" или какой файл не может прочитать.
```bash
    strace -p 1234  # Подключиться к работающему процессу и смотреть его логи
```
