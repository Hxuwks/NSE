**Подсеть (Subnetwork, Subnet)** — это логическое разделение единой IP-сети на более мелкие, независимые сегменты. Каждая подсеть представляет собой отдельный широковещательный домен и имеет свой уникальный адрес сети.

Изначально IP-адреса делились на классы (A, B, C) с фиксированными масками. Однако такой подход был неэффективен: компания с 300 компьютерами получала целый класс B (/16) с 65 534 адресами, что приводило к колоссальной трате адресного пространства. Современный подход — **бесклассовая адресация (CIDR)** — позволяет "нарезать" сеть на подсети любого размера, используя маски произвольной длины.

> **Ключевая идея подсетей:** Вы берёте одну большую сеть (например, `192.168.0.0/16`) и делите её на несколько меньших сетей (например, `192.168.1.0/24`, `192.168.2.0/24` и т.д.). Это позволяет организовать структуру сети, уменьшить широковещательный трафик и повысить безопасность.

### 1. Фундаментальные понятия

Прежде чем перейти к делению сетей, необходимо чётко усвоить три базовых понятия, которые определяют любую подсеть:

| Понятие | Определение | Пример (для /24) |
| :--- | :--- | :--- |
| **Адрес сети** (Network Address) | Минимальный адрес в подсети. Все биты узла (host) равны нулю. Используется для идентификации подсети в таблицах маршрутизации. | `192.168.1.0` |
| **Широковещательный адрес** (Broadcast Address) | Максимальный адрес в подсети. Все биты узла равны единице. Используется для отправки пакетов всем устройствам в подсети одновременно. | `192.168.1.255` |
| **Первый/последний доступный адрес** | Адреса, которые можно назначать сетевым интерфейсам (хостам). | `192.168.1.1` и `192.168.1.254` |
| **Количество доступных хостов** | Общее количество адресов в подсети минус 2 (адрес сети и широковещательный адрес). | `256 - 2 = 254` |

**Формула для расчёта количества адресов:**
```
Количество адресов = 2^(32 - длина_префикса)
Количество доступных хостов = 2^(32 - длина_префикса) - 2
```

### 2. Наиболее распространённые маски подсетей

В реальной практике чаще всего используются всего несколько масок. Понимание их "размера" и области применения — основа основ.

| Префикс (CIDR) | Маска (десятично) | Кол-во адресов | Кол-во хостов | Типичное применение |
| :--- | :--- | :--- | :--- | :--- |
| **/8** | `255.0.0.0` | 16 777 216 | 16 777 214 | Крупные корпоративные сети, дата-центры. Приватный диапазон `10.0.0.0/8`. |
| **/16** | `255.255.0.0` | 65 536 | 65 534 | Средние сети (университеты, большие офисы). Приватный диапазон `172.16.0.0/12` состоит из 16 таких сетей. |
| **/24** | `255.255.255.0` | 256 | **254** | **Самый популярный размер.** Домашние сети, малые офисы, сегменты в дата-центрах. Приватный диапазон `192.168.0.0/16` состоит из 256 таких сетей. |
| **/29** | `255.255.255.248` | 8 | **6** | Микро-сегменты для подключения нескольких серверов. |
| **/30** | `255.255.255.252` | 4 | **2** | **Точка-точка (P2P).** Используется для соединения двух маршрутизаторов. Максимальная эффективность использования адресов. |
| **/31** | `255.255.255.254` | 2 | **2** | Современный стандарт для P2P-линков (RFC 3021). Не требует широковещательного адреса. Используется в крупных сетях провайдеров для экономии адресов. |
| **/32** | `255.255.255.255` | 1 | **1** | Обозначение одного конкретного хоста (loopback-интерфейс маршрутизатора). |

### 3. Детальный разбор самых важных масок

#### 3.1. /24 — стандарт для локальных сетей

Это "золотой стандарт" для подавляющего большинства сетей. Маска `/24` означает, что под сеть отводится 24 бита (первые три октета), а под узлы — 8 бит (последний октет, от 0 до 255).

**Структура:** `Сеть.Сеть.Сеть.Узел`

**Пример:** `192.168.1.0/24`
- **Адрес сети:** `192.168.1.0`
- **Диапазон адресов для хостов:** `192.168.1.1` – `192.168.1.254`
- **Широковещательный адрес:** `192.168.1.255`
- **Количество хостов:** 254

**Внутреннее представление:**
```
IP-адрес:   11000000.10101000.00000001.00001010  (192.168.1.10)
Маска /24:  11111111.11111111.11111111.00000000  (255.255.255.0)
            └─────────── СЕТЬ (24 бита) ──────────┘└─ УЗЕЛ (8 бит) ─┘
```

**Назначение:** Любая домашняя сеть Wi-Fi, офисный сегмент, VLAN. Достаточно 254 адресов для большинства локальных сетей.

#### 3.2. /30 — точка-точка (Point-to-Point)

Это специальная маска, которая даёт всего 2 доступных адреса. Она используется исключительно для соединения двух устройств (обычно двух маршрутизаторов) напрямую.

**Структура:** `Сеть.Сеть.Сеть.Подсеть (2 бита для узлов)`

**Пример:** `10.0.0.0/30`
- **Адрес сети:** `10.0.0.0`
- **Первый хост:** `10.0.0.1` (обычно один из маршрутизаторов)
- **Второй хост:** `10.0.0.2` (второй маршрутизатор)
- **Широковещательный адрес:** `10.0.0.3` (не используется, так как в P2P-линке он не нужен)
- **Количество хостов:** 2

**Внутреннее представление:**
```
IP-адрес:   00001010.00000000.00000000.00000001  (10.0.0.1)
Маска /30:  11111111.11111111.11111111.11111100  (255.255.255.252)
            └─────────── СЕТЬ (30 бит) ───────────┘└ УЗЕЛ (2 бита) ─┘
```

**Назначение:** Соединение маршрутизаторов в магистральных сетях провайдеров, где важна экономия публичных IP-адресов. Вместо того чтобы тратить целую сеть `/24` на два маршрутизатора, используется `/30`, что позволяет сэкономить 254 адреса.

**Важно:** Сеть `/30` имеет всего 4 адреса: адрес сети, два адреса для хостов и широковещательный адрес. Это максимально эффективное использование адресного пространства для линка.

#### 3.3. /16 — крупные сети

Эта маска выделяет под сеть 16 бит (первые два октета), а под узлы — 16 бит (последние два октета). Это очень большая сеть, которая может обслуживать десятки тысяч устройств.

**Структура:** `Сеть.Сеть.Узел.Узел`

**Пример:** `172.16.0.0/16`
- **Адрес сети:** `172.16.0.0`
- **Диапазон адресов для хостов:** `172.16.0.1` – `172.16.255.254`
- **Широковещательный адрес:** `172.16.255.255`
- **Количество хостов:** 65 534

**Внутреннее представление:**
```
IP-адрес:   10101100.00010000.00000000.00000001  (172.16.0.1)
Маска /16:  11111111.11111111.00000000.00000000  (255.255.0.0)
            └────── СЕТЬ (16 бит) ───────┘└────── УЗЕЛ (16 бит) ──────┘
```

**Назначение:** В реальной практике сеть `/16` почти всегда делится на множество подсетей `/24`. Однако в некоторых случаях она используется как единая сеть:
- Приватный диапазон `10.0.0.0/8` может быть разбит на сотни сетей `/16` для разных филиалов компании.
- Внутри дата-центра может существовать сеть `/16` для управления инфраструктурой.

### 4. Как "нарезать" сеть на подсети (базовый подход)

Процесс деления большой сети на несколько меньших называется **subnetting**. Рассмотрим на примере: у нас есть сеть `192.168.0.0/16` (65 534 хоста), и мы хотим разделить её на 256 подсетей по 254 хоста в каждой (то есть на сети `/24`).

**Шаг 1: Определить, сколько бит нужно "занять" у узла, чтобы получить нужное количество подсетей.**
- Мы хотим получить 256 подсетей.
- Нужно найти такое количество бит `n`, чтобы `2^n >= 256`. `2^8 = 256`.
- Значит, мы забираем 8 бит из части узла и отдаём их под сеть.

**Шаг 2: Определить новую маску.**
- Исходная маска: `/16` (16 бит под сеть, 16 бит под узлы).
- Забираем 8 бит у узлов: новая маска = `/24` (16 + 8 = 24 бита под сеть, 8 бит под узлы).

**Шаг 3: Определить адреса новых подсетей.**
- Мы забираем 8 бит, значит, эти биты будут меняться от `00000000` до `11111111` в двоичном виде, что соответствует десятичным значениям от 0 до 255.
- Это означает, что мы получим 256 подсетей:
    - `192.168.0.0/24`
    - `192.168.1.0/24`
    - `192.168.2.0/24`
    - ...
    - `192.168.255.0/24`

**Наглядная схема деления:**

```text
Исходная сеть: 192.168.0.0/16
┌─────────────────────────────────────────────────────────────┐
│  СЕТЬ (16 бит: 192.168)  │  УЗЛЫ (16 бит: 0.0 ─ 255.255)    │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  СЕТЬ (24 бита: 192.168.X)  │  УЗЛЫ (8 бит: 0 ─ 255)        │
└─────────────────────────────────────────────────────────────┘
                            │
        ┌───────────────────┼───────────────────┬─────────────┐
        ▼                   ▼                   ▼             ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 192.168.0.0  │    │ 192.168.1.0  │    │ 192.168.2.0  │    │192.168.255.0 │
│    /24       │    │    /24       │    │    /24       │    │    /24       │
│ (254 хоста)  │    │ (254 хоста)  │    │ (254 хоста)  │    │ (254 хоста)  │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

### 5. Таблица быстрого расчёта для популярных масок

Эта таблица помогает быстро оценить "размер" сети и её границы, не выполняя сложных вычислений.

| Префикс | Маска | Количество адресов | Количество хостов | Шаг подсети (инкремент) | Пример сети |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **/24** | `255.255.255.0` | 256 | 254 | 1 (последний октет) | `192.168.1.0` |
| **/25** | `255.255.255.128` | 128 | 126 | 128 | `192.168.1.0`, `192.168.1.128` |
| **/26** | `255.255.255.192` | 64 | 62 | 64 | `192.168.1.0`, `.64`, `.128`, `.192` |
| **/27** | `255.255.255.224` | 32 | 30 | 32 | `192.168.1.0`, `.32`, `.64`, `.96`, `.128`... |
| **/28** | `255.255.255.240` | 16 | 14 | 16 | `192.168.1.0`, `.16`, `.32`, `.48`... |
| **/29** | `255.255.255.248` | 8 | 6 | 8 | `192.168.1.0`, `.8`, `.16`, `.24`... |
| **/30** | `255.255.255.252` | 4 | 2 | 4 | `192.168.1.0`, `.4`, `.8`, `.12`... |

**Как пользоваться таблицей:**
- Если у вас маска `/27`, то каждая подсеть занимает 32 адреса.
- Адреса сетей будут начинаться с `...0`, `...32`, `...64`, `...96`, `...128` и т.д.
- Первый доступный адрес — это адрес сети + 1. Широковещательный — это адрес следующей сети минус 1.

**Пример для `/27`:**
- Сеть: `192.168.1.0/27`
    - Адрес сети: `192.168.1.0`
    - Первый хост: `192.168.1.1`
    - Последний хост: `192.168.1.30`
    - Широковещательный: `192.168.1.31`
- Следующая сеть: `192.168.1.32/27`
    - Адрес сети: `192.168.1.32`
    - Первый хост: `192.168.1.33`
    - И так далее.

### 6. Почему маска важна: сравнение разных масок для одного адреса

Один и тот же IP-адрес с разными масками принадлежит к разным сетям и имеет разный широковещательный адрес. Это критично для понимания.

**Пример:** Адрес `10.1.2.3`

| Маска | Адрес сети | Широковещательный адрес | Кто находится в одной сети с этим хостом? |
| :--- | :--- | :--- | :--- |
| `/8` | `10.0.0.0` | `10.255.255.255` | Все устройства с адресами `10.x.x.x` (огромная сеть) |
| `/16` | `10.1.0.0` | `10.1.255.255` | Только устройства с адресами `10.1.x.x` |
| `/24` | `10.1.2.0` | `10.1.2.255` | Только устройства с адресами `10.1.2.x` |
| `/30` | `10.1.2.0` | `10.1.2.3` | **Некорректно!** Адрес `10.1.2.3` является широковещательным для `/30` и не может быть назначен хосту. |

**Вывод:** Маска определяет границы сети. Два устройства с одинаковым IP-адресом, но с разными масками, могут не "видеть" друг друга, потому что они будут считать, что находятся в разных широковещательных доменах.
